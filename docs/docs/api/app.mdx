# ðŸ“± App Info API

The App Info API provides information about the Pika application, including system status, available features, version information, and application lists. This API is useful for getting application metadata and checking system health.

## ðŸ“¡ Base URL

All app info endpoints are prefixed with:
```
/wp-json/pika/v1/app
```

## ðŸš€ Endpoints

### App Info Service

```typescript
// services/app.service.ts
export class AppService {
  async getAppInfo(): Promise<AppInfoResponse> {
    const response = await api.get('/app/info');
    return response.data.data;
  }

  async getAppLists(): Promise<AppListsResponse> {
    const response = await api.get('/app/lists');
    return response.data.data;
  }

  async getSystemStatus(): Promise<SystemStatusResponse> {
    const response = await api.get('/app/status');
    return response.data.data;
  }

  async getFeatures(): Promise<FeaturesResponse> {
    const response = await api.get('/app/features');
    return response.data.data;
  }

  async checkFeatureAvailability(featureName: string): Promise<boolean> {
    const features = await this.getFeatures();
    return features.features[featureName]?.enabled || false;
  }
}
```

## ðŸ’¡ Best Practices

   - Show loading states during checks

   - Offer help for system issues
